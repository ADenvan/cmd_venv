# Примеры использования навыка `project-health-check`

## Пример 1. Базовый запрос health check

**Запрос пользователя**

> Посмотри, пожалуйста, на репозиторий и скажи, насколько он в порядке. Нужен быстрый health check.

**Ожидаемые действия агента**

1. Определить тип проекта по ключевым файлам (Python/JS и т.п.).
2. Оценить структуру (отдельные папки для кода и тестов, наличие конфигов).
3. Проверить наличие файлов зависимостей и lock‑файлов.
4. Найти и по возможности запустить базовые тесты и линтеры (если пользователь не возражает).
5. Сформировать отчёт в формате из раздела `Report format` навыка.

**Пример отчёта**

```markdown
## Project health summary

**Общий вывод:** структура проекта в целом понятна, есть базовое разделение кода и тестов, но не хватает lock‑файла и автоматического запуска линтера.

### Сильные стороны
- Код и тесты разнесены по разным директориям.
- Есть единый файл зависимостей `pyproject.toml`.

### Проблемы и риски
- [Важно] Отсутствует lock‑файл, версии зависимостей могут «плыть» между окружениями.
- [Минорно] Линтер не интегрирован в удобную команду, требуется ручной запуск.

### Рекомендованные следующие шаги
1. Добавить lock‑файл (например, зафиксировать зависимости через менеджер, используемый в проекте).
2. Настроить команду запуска линтера (например, через `make` или скрипт в `pyproject.toml`).
3. Добавить краткую документацию по запуску тестов и проверок в `README`.
```

## Пример 2. Проверка перед передачей проекта

**Запрос пользователя**

> Мы собираемся отдать этот репозиторий другой команде. Посмотри, что стоит улучшить, чтобы проект выглядел более «здоровым».

**Ожидаемые действия агента**

1. Выполнить те же шаги, что и в базовом health check.
2. Особое внимание уделить:
   - наличию понятного `README`;
   - описанию, как запускать проект и тесты;
   - единообразию структуры и инструментов.
3. В отчёте отдельно выделить, что важно сделать до передачи проекта.

**Пример дополнительных рекомендаций**

- Добавить раздел «Как начать» в `README` (установка, запуск, тесты).
- Убедиться, что команды для тестов и линтеров описаны и работают на чистом окружении.
